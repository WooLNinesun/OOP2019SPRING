ATM

Deadline

2019-06-24 23:59:59.

Description

Write a program that can serve as a simple ATM (Automated Teller Machine ).
This simple ATM only provides service of withdrawals.
As ATMs in real world, a user can withdraw money from this simple ATM only when the balance of his/her account is sufficient. Moreover, withdrawals are restricted to be in thousands, with one-thousand dollar bills provided only.

You need to contruct a class named Simple_ATM_Service Simple_ATM_Service with implementing given interface ATM_Service ATM_Service .
Interface ATM_Service ATM_Service prepares some base function of ATM.
For our simple ATM, more specifically, checkBalance checkBalance should help checking whether balance in user's account is sufficient, if not, throws an exception named ATM_Exception ATM_Exception with type of " BALANCE_NOT_ENOUGH BALANCE_NOT_ENOUGH "; isValidAmount isValidAmount checks if amount of money can be divided by 1000, if not, throws an exception named ATM_Exception ATM_Exception with type of " AMOUNT_INVALID AMOUNT_INVALID "; withdraw withdraw first calls checkBalance checkBalance and then calls isValidAmount isValidAmount to check if it is a valid operation.If valid, simple ATM will debit for amount of money the user specified ,and balance of user's account will also be updated. withdraw withdraw has to catch the exceptions raised by checkBalance checkBalance and isValidAmount isValidAmount , and use getMessage getMessage defined in ATM_Exception ATM_Exception to show the exception information.
At the end of withdraw withdraw function, it will always show updated balance in user's account in format of " updated balance : XXX updated balance : XXX ", no matter whether the user withdraws the money successfully or not.

To fulfill the whole functionality, you will needs another class named  ATM_Exception  ATM_Exception extending Exception.
It contains an enumerated type ExceptionTYPE ExceptionTYPE which includes two kinds of exception. To record the detail of exception raised, we need a private variable excptionCondition excptionCondition with the type of ExceptionTYPE ExceptionTYPE we just defined and this variable would be set by constructor. For ATM to get the imformation of exception raised, use getMessage getMessage .

Account Account class has already been done for you, you just need to copy the code provided in Required Files Required Files section of this page.

NOTICE: NOTICE:
Do not write multiple try-catch blocks in withdraw withdraw . You just need to catch/print the first exception raised.
